# Legal Documents Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `feature/legal` in `:shared` with in-app WebView for Privacy Policy and Terms of Service inside modal bottom sheets, external-browser fallback, and host-supplied URLs — independent of `NavHost` routes.

**Architecture:** Presentation-only feature package (`api`/`impl`); `LegalDocumentType` + `LegalDocumentBottomSheetHost`; local visibility state at host call site; platform `expect/actual` for WebView and `rememberOpenUrlExternally`.

**Tech Stack:** Kotlin Multiplatform, Compose Multiplatform, Material3 `ModalBottomSheet`, Android WebView, iOS WKWebView.

**Spec:** [`docs/superpowers/specs/2026-06-25-legal-design.md`](../specs/2026-06-25-legal-design.md)
